Gap Analysis Assignment


For my final project in my Web Application Programming course, I was asked to pick a web related technology or framework that we did not cover in class and study it in depth and then explain how it works to the class in a PowerPoint presentation. I chose the multi-platform mobile development Framework Apache Cordova. In order to get a better understanding of Cordova and how it works, I studied the documentation as well as watched educational videos on YouTube. I learned the process of how everyday standard web languages like HTML, CSS, and JS can be used to create multi platform applications with ease.


As part of my process of learning about Cordova, I created a basic multi platform application. In order to develop the app, I followed along with the instruction in the documentation for Cordova. A lot of frameworks offer a tutorial on how to get your first app up and running, so that is what I did to create my app. The app itself does not do much as it is pretty much just displaying the default Cordova welcome page, but I was at least able to get a functioning app that could run on multiple different platforms all based off web languages.


My original intent when creating my app was to have it do some sort of function or pull from an API just for demonstrational purposes, but I found a lot of my time went into the research portion of things and I ran out of time to do anything with the app beyond the end of the tutorial. Since Cordova is a framework for developing multi-platform application - there was a lot things to learn about in regards to setting up my development environment. For example, I had to install SDKs for both IOS and Android as well as Java. I also had trouble getting requirements to run the android emulator from Android Studio on my computer at home.


During my learning process, I got an idea of how a lot of different things work in Cordova. I learned how things are divided by application wrappers and how the wrappers interact with eachother. One of the most interesting things that I learned about was working with plugins. Plugins allow you to access device level functionality from within your app such as camera, battery, network, etc. I also learned quite a bit about the Cordova CLI and how it works similar to the way Laravel's Artisan CLI works to scaffold required files quickly and carry out other useful functionality that you want want to do.


I was able to get some experience working with the Android Studio IDE, which is really cool because it comes with a lot of useful functionality such as interfaces for adding device emulators that you can preview your application on as well as adding project requirements such as platform specific development kits. Overall I enjoyed learning about Cordova and how it can open up so many opportunities in the mobile world for me as a web developer. It is absolutely something that I plan on working with in the future and I will more than likely see it in the indsutry at some point.
